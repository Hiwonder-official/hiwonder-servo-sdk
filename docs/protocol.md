# HX-30HM 总线舵机通信协议

> 适用型号：幻尔 HX-30HM 磁编码总线舵机  
> 协议版本：1.0（2025-09-25）  
> 固件版本：2.43

**[English](protocol_en.md) | 中文**

---

## 1. 概要

HX-30HM 采用 **ARM 32 位单片机**为主控，位置感应采用 360° 12 位精度磁铁感应方案。

总线拓扑允许多个舵机挂接，每个舵机拥有唯一 **ID（0–253）**。  
**ID 254（0xFE）** 为广播 ID，所有舵机均接收指令，但除 PING 外不返回应答包。

### 串口参数

| 参数 | 值 |
|------|----|
| 接口 | UART（单线半双工）|
| 波特率（默认）| **1 000 000 bps** |
| 可选波特率 | 500 000 / 250 000 / 128 000 / 115 200 / 76 800 / 57 600 / 38 400 |
| 数据位 | 8 |
| 停止位 | 1 |
| 校验 | 无 |
| 帧长 | 10 bit（1 起始 + 8 数据 + 1 停止）|

---

## 2. 数据帧格式

### 2.1 指令包（主机 → 舵机）

```
┌────────┬────────┬────────┬─────────────┬─────────────────────────┬──────────┐
│ 0xFF   │ 0xFF   │  ID    │   Length    │  Instruction + Params   │ CheckSum │
└────────┴────────┴────────┴─────────────┴─────────────────────────┴──────────┘
```

| 字段 | 字节数 | 说明 |
|------|--------|------|
| Header | 2 | 固定 `0xFF 0xFF`，标识数据包开始 |
| ID | 1 | 目标舵机 ID（0x00–0xFD；0xFE = 广播）|
| Length | 1 | `N + 2`，N 为参数个数 |
| Instruction | 1 | 指令码，见第 3 节 |
| Parameter 1…N | N | 指令附加参数 |
| CheckSum | 1 | `~(ID + Length + Instruction + P1 + … + PN) & 0xFF` |

> **校验和**：将括号内所有字节求和，取最低字节，再按位取反。

### 2.2 应答包（舵机 → 主机）

```
┌────────┬────────┬────────┬────────────┬────────┬─────────────────┬──────────┐
│ 0xFF   │ 0xFF   │  ID    │   Length   │ ERROR  │  Parameter 1…N  │ CheckSum │
└────────┴────────┴────────┴────────────┴────────┴─────────────────┴──────────┘
```

| 字段 | 说明 |
|------|------|
| ERROR | 舵机当前错误状态（0 = 正常）|
| Parameter | 仅读指令时含返回数据 |

### 2.3 字节序

所有 **多字节数值均为小端（Little-Endian）**，即低位字节在前、高位字节在后。

---

## 3. 指令类型

| 指令名称 | 指令码 | 参数长度 | 说明 |
|----------|--------|----------|------|
| PING | `0x01` | 0 | 查询舵机工作状态 |
| READ DATA | `0x02` | 2 | 从内存表读取数据 |
| WRITE DATA | `0x03` | ≥ 1 | 写数据到内存表 |
| REG WRITE | `0x04` | ≥ 2 | 异步写（写入缓冲区，等待 ACTION）|
| ACTION | `0x05` | 0 | 触发执行 REG WRITE 缓冲的数据 |
| RESET | `0x06` | 0 | 复位控制表为出厂值 |
| SYNC READ | `0x82` | ≥ 3 | 同步读多个舵机 |
| SYNC WRITE | `0x83` | ≥ 2 | 同步写多个舵机 |

---

## 4. 各指令详解

### 4.1 PING — 查询状态

**指令帧格式**

```
FF FF  <ID>  02  01  <CheckSum>
```

**示例**：查询 ID=1 的舵机

```
发送：FF FF 01 02 01 FB
返回：FF FF 01 02 00 FC
```

返回包中 ERROR=0x00 表示舵机正常。

---

### 4.2 READ DATA — 读数据

**指令帧格式**

```
FF FF  <ID>  04  02  <StartAddr>  <DataLen>  <CheckSum>
```

| 参数 | 说明 |
|------|------|
| StartAddr | 读取起始地址（1 字节）|
| DataLen | 读取字节数（1 字节）|

**示例**：读取 ID=1 的当前位置（地址 0x38，2 字节）

```
发送：FF FF 01 04 02 38 02 BE
返回：FF FF 01 04 00 18 05 DD
```

返回数据：低字节 0x18，高字节 0x05 → 位置值 = 0x0518 = **1304**

---

### 4.3 WRITE DATA — 写数据

**指令帧格式**

```
FF FF  <ID>  <N+3>  03  <StartAddr>  <Data1> … <DataN>  <CheckSum>
```

**示例 1**：设置任意舵机 ID 为 1（广播发送，地址 0x05）

```
FF FF FE 04 03 05 01 F4
```

> 注意：修改 NVS 区域前需先关闭写保护锁（地址 0x37 置 0）。

**示例 2**：控制 ID=1 以速度 1000 步/秒运动到位置 2048

目标位置 0x2A 起写 6 字节：位置(2字节) + 时间(2字节) + 速度(2字节)

```
FF FF 01 09 03 2A 00 08 00 00 E8 03 D5
```

返回：`FF FF 01 02 00 FC`（ERROR=0，指令执行中）

---

### 4.4 REG WRITE — 异步写

与 WRITE DATA 格式相同，指令码改为 `0x04`。  
收到该指令后舵机将数据存入缓冲区，并将 `Registered Instruction` 寄存器置 1，  
直到收到 ACTION 指令后才真正执行。

**用途**：在多舵机场景中，先逐一发送 REG WRITE，再发送一条 ACTION 让所有舵机同时动作。

---

### 4.5 ACTION — 执行异步写

```
FF FF FE 02 05 FA
```

使用广播 ID（0xFE），无返回数据。所有在总线上的舵机同步执行各自缓冲的指令。

---

### 4.6 RESET — 复位

将内存控制表恢复出厂值。

```
FF FF <ID> 02 06 <CheckSum>
```

**示例**：复位 ID=1 的舵机

```
发送：FF FF 01 02 06 F6
返回：FF FF 01 02 00 FC
```

---

### 4.7 SYNC WRITE — 同步写

一条指令同时写多个舵机，写入地址和数据长度必须相同。

**指令帧格式**

```
FF FF  FE  <Length>  83  <StartAddr>  <DataLen(L)>
  <ID1>  <D1_1> … <D1_L>
  <ID2>  <D2_1> … <D2_L>
  …
  <CheckSum>
```

其中 `Length = (L + 1) × N + 4`（N = 舵机数）

**示例**：同步控制 ID1–ID4 运动到位置 2048，速度 1000，时间 0

```
FF FF FE 20 83 2A 06
  01 00 08 00 00 E8 03
  02 00 08 00 00 E8 03
  03 00 08 00 00 E8 03
  04 00 08 00 00 E8 03
58
```

使用广播 ID，无返回数据。

---

### 4.8 SYNC READ — 同步读

一条指令查询多个舵机，查询地址和数据长度必须相同。

**指令帧格式**

```
FF FF  FE  <N+4>  82  <StartAddr>  <DataLen>
  <ID1>  <ID2>  …  <IDN>
  <CheckSum>
```

舵机按 ID 顺序依次返回各自的应答包。

**示例**：查询 ID1、ID2 的当前位置+速度+负载+电压+温度（地址 0x38，8 字节）

```
发送：FF FF FE 06 82 38 08 01 02 36

返回 ID1：FF FF 01 0A 00  00 08  00 00  00 00  79  1E  55
返回 ID2：FF FF 02 0A 00  FF 07  00 00  00 00  77  23  53
```

## 5. 带符号数据的表示

对于带符号数据（如速度、位置校正值），当值为负时使用**二进制原码**表示，最高位为符号位。

**示例**：以 BIT15 为符号位，16 位数据

| 值 | 二进制（BIT15…BIT0）|
|----|---------------------|
| +2000 | `0 000011111010000` |
| -2000 | `1 000011111010000` |

---

## 6. 校验和计算示例

```python
def calc_checksum(packet: list[int]) -> int:
    # packet = [ID, Length, Instruction, P1, ..., PN]
    return (~sum(packet)) & 0xFF
```

---

## 7. 通信时序建议

- 每次发送指令后建议等待至少 **1 ms** 再读取应答。
- 广播指令（ID=0xFE）无应答，无需等待。
- 多舵机同步控制推荐优先使用 **SYNC WRITE**，其实时性优于 REG WRITE + ACTION。
